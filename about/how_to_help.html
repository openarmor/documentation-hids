How to start helping with the project?¶

OpenArmor is maintained by a small group of people from around the world. If you wish to get involved, there are multiple ways to do so.

Testing OpenArmor:¶

The easiest way of getting involved with OpenArmor is by helping testing it.

Translating OpenArmor:¶

Translating OpenArmor is easy. We already support many languages, but new ones are more than welcome and fixes for the ones we have already too.

After you download OpenArmor and untar/ungzip it, you will find the messages to be translated inside the etc/templates directory:

$ ls -la etc/templates/
br
cn
en
..
tr

Inside each directory you will find a messages.txt and language.txt file and a messages/ and errors/ directory. If you are fixing one of the languages we already have, just modify them and send the changes to us.

If you want to support a new language, just copy the English one to your country code and start translating:

$ cp -pr en br
$ vim br/messages.txt

Documenting OpenArmor:¶

The OpenArmor documentation is hosted in the OpenArmor-docs github repository. Issues and pull requests can be submitted on the site. Emails containing details of issues can also be sent to the OpenArmor-list google group.

Development of OpenArmor:¶

The OpenArmor code is hosted in the OpenArmor-hids github repository. Issues and pull requests can be submited on the site.

Contributing to the development of OpenArmor’s code base will most likely require knwoledge of C, and will take time learning how the internals work.
