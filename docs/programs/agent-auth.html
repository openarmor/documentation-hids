agent-authÂ¶

The agent-auth program is the client application used with OpenArmor-authd to automatically add agents to an OpenArmor manager.

Warning

By default there is no authentication or authorization involved in this transaction, so it is recommended that this daemon only be run when a new agent is being added.

Optional Server Authentication - client sideÂ¶

agent-auth can verify that the server itâs connecting to presents a valid X.509 certificate when requesting a key. This is optional and is only useful if hosts in your environment have access to the root certificate of the CA that signed the certificate presented by OpenArmor-authd. If server certificate verification is desired then the relevant CA certificate must be loaded with the -v option, then if the server does not present a valid certificate the agent will not be allocated a key.

A certificate presented by the server may be found to be invalid for the following reasons:

  • It was not signed by the specified CA.

  • It doesnât contain the IP address or hostname given with the -m option in the subjectâs common name field or a subject alternative name extension field.

  • It is expired.

While server authentication is optional it is highly recommended that it be used if possible when running OpenArmor-authd and agent-auth.

Optional Client AuthenticationÂ¶

agent-auth can present its own certificate to the server for verification. This is mandatory if OpenArmor-authd was run with the -v option and optional otherwise. This is only useful if hosts in your environment are assigned certificates when theyâre provisioned (or at some point before being added to OpenArmor). Use the -x and -k options to load a certificate and private key.
